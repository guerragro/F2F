Советы по коду: если у вас есть массив и информация с него будет где то использоваться и меняться. Используйте get and set и храните его в сервисе. ! Но если, это данные не будут менять, то и не стоит использовать лишний раз get and set service.ts public _wares: any; get wares() { return _this.wares }; set wares(value) { this._wares = value } если в запросе с вашего сервиса у вас приходит какая-то информация, допустим товар, делайте обязательно проверки wares.length !== 0, проверяем на длину массив wares instanceof Array, проверям пришел ли нам массив

NgClass [ngClass]="{ verdanaFont: isVerdana }" [class.first]="this.test === true" где verdanaFont and first, это класс, а isVerdana and test это переменная исходя из которой будет применяться класс Если нужно, что к блоку применялся какой-то из классов [class.first]="test === true" [class.second]="test === false" [ngClass]="{first: test === true, second: test === false}" где first and second это классы Вы так же можете использовать *ngIf *ngIf="test === true" class="first" *ngIf="test !== true" class="second"
NgModel Двухсторонняя привязка [(NgModel)]="переменная" wares: string = '' [(NgModel)]="wares" Как правильно применяется в связке с input.
